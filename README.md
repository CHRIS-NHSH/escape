# escape — 機關範例庫

校園密室／闖關活動用的互動機關範例庫。首頁彙整各式機關，每個機關都是**獨立的單檔 HTML 網頁**，繁體中文介面、手機／平板可直接觸控操作，離線也能跑。

線上版（GitHub Pages）：`https://chris-nhsh.github.io/escape/`

---

## 結構

- `index.html` — 首頁（範例庫）。長按右上角齒輪 → 進首頁後台，可新增／修改／刪除／排序機關、匯出入設定、設定程式碼分享。
- `teach/index.html` — 教學頁：**用 AI 做出這些機關**（給學生自學，含「規格 → 提示詞產生器」與簡報模式）。
- `template/index.html` — 空白機關起手骨架（共用外殼已備好，找 `✏️` 標記填入玩法）。

## 共用功能（每個機關都有）

- **後台**：長按右上角齒輪 0.6 秒 → 輸入後台密碼（**預設 `9633`**）→ 進入設定。玩家看不到後台。
- **揭曉卡**：過關後顯示線索文字／圖片（圖片可點擊放大）。
- **音效**：Web Audio，可在後台關閉。
- **記憶**：設定存在該裝置的瀏覽器（localStorage）。
- **產生分享連結**：後台可把「目前設定」編碼成一條獨立網址（`?c=...`）。同一個機關可重複用於不同遊戲——各自產生一條連結即可，不需後端。
  - 注意：分享連結內含設定（含答案），適合課堂使用；要隱藏答案才需要後端。
  - 上傳的圖片會讓網址過長，分享時請改用「圖片網址」。

## 需要注意的機關（語音相關）

- **No.10 電話撥號 / No.11 密室收音機**：語音用裝置的「語音合成（TTS）」直接把文字唸出來，品質與有無中文聲音**因裝置而異**；要最穩就在後台改放「音檔網址」的預錄語音。收音機若語音沒出聲，會自動顯示字幕當答案。
- **No.12 咒語**：使用瀏覽器「語音辨識（Web Speech API）」。
  - 必須在 **https 網站**上執行（GitHub Pages 沒問題），並允許**麥克風權限**。
  - 支援度：**Chrome／Edge／Android 最佳；Firefox 不支援；iOS Safari 視版本而定**。（多數線上預覽視窗無法用麥克風，請到實際部署網址、用 Chrome 測。）
  - 不支援或權限被拒時，畫面會自動提供**「改用打字」**後援，任何裝置都能過關。

## 範本：做一個新機關

`template/index.html` 已備好全部共用外殼（齒輪／後台／揭曉卡／音效／分享連結）。複製後找 `✏️` 標記，填入四個部分：設定（cfg）、正規化（normalize）、畫面建立（build）、過關判定（solve）。

也可以參考教學頁 `teach/`：先寫「設計規格」，再用「提示詞產生器」把需求交給 AI 實作。

---

## 新增機關到首頁

1. 在 `<名稱>/index.html` 放該機關的單檔網頁。
2. 首頁後台「新增機關」，連結填 `<名稱>/`，狀態設「可遊玩」。
3. 要讓所有人都看到：用首頁後台「匯出設定」，把內容貼回 `index.html` 的 `DEFAULT_HUB` 後重新上傳。

## 上傳／部署（GitHub Pages）

- 用 GitHub 網頁：Add file → Upload files，把整個資料夾拖進去；或用 git。
- 啟用 Pages：Settings → Pages → 選 `main /(root)`。
- 部署後：`https://chris-nhsh.github.io/escape/`
- 上線後可到各機關後台把預設密碼 `9633` 改成自己的，並在首頁後台設定程式碼分享密碼。
